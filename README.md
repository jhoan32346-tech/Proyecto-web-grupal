# 📌 Actividad Grupal: Proyecto Colaborativo en GitHub (INFOTEP)

## Descripción del Proyecto
Este repositorio corresponde a la **Actividad Grupal**, cuyo propósito es simular un entorno real de desarrollo colaborativo utilizando la plataforma GitHub. 

Durante el desarrollo de esta actividad, sé aplicaron practicas como control de versiones, trabajo en equipo y gestión de cambios mediante el uso de ramas, commits y Pull Requests.

## Objetivos

### Objetivo General
Aplicar el uso de herramientas de control de versiones mediante un proyecto colaborativo en GitHub.

### Objetivos Específicos
* Crear y configurar un repositorio compartido.
* Gestionar colaboradores dentro del proyecto.
* Implementar un flujo de trabajo con ramas.
* Realizar commits y seguimiento de cambios.
* Utilizar **Pull Requests** para integración de código.
* Ejecutar revisiones de código entre compañeros.

## Actividades Realizadas

### 🔹 Como Grupo
1. Creación de un repositorio de ejemplo en GitHub.
2. Agregación de los integrantes como colaboradores.
3. Definición de un flujo de trabajo automatizado con **GitHub Actions**.

### 🔹 De Forma Individual
Cada integrante del grupo realizó las siguientes tareas:
* Clonar el repositorio.
* Crear una rama personal (`feature/nombre`).
* Realizar modificaciones en archivos del proyecto.
* Registrar cambios mediante **commits**.
* Subir cambios al repositorio (**push**).
* Crear un **Pull Request**.
* Revisar el Pull Request de otro compañero.
* Aprobar e integrar los cambios a la rama principal (`main`).
---

## Flujo de Trabajo
El proyecto se desarrolló siguiendo un flujo estructurado de control de versiones:

> **main** → **rama personal** → **commit** → **push** → **pull request** → **revisión** → **merge**

## GitHub Actions
Se implementó un flujo básico de automatización utilizando GitHub Actions, con el fin de:
* Validar cambios realizados en el repositorio.
* Mantener un control básico sobre las integraciones.
